## Deploy Workflow

```bash
./deploy-ha-dashboards.sh
```

Deploys all dashboards (or a single one — see script flags). Touch ID required.

### Full workflow for a dashboard change

1. Edit the YAML file in `ha-dashboards/`
2. Review the diff: `git diff`
3. Run: `./deploy-ha-dashboards.sh`
4. Hard-refresh the browser on the affected dashboard (`Cmd+Shift+R`)
5. Commit: `git add ha-dashboards/ && git commit -m "feat: ..."`

### What deploy-ha-dashboards.sh does

1. Reads HA token from 1Password CLI (`op read`)
2. SCPs `ha-dashboards/*.yaml` → `/config/dashboards/` on HA
3. YAML-mode dashboards are read from disk on each browser request — hard-refresh (`Cmd+Shift+R`) picks up changes immediately, no HA restart needed

## Adding a New Dashboard

New dashboards require two steps and a **full HA restart** (unlike card changes, which only need a Lovelace reload):

1. Create the YAML file in `ha-dashboards/<slug>.yaml`
2. Add an entry to the `lovelace: dashboards:` block in `configuration.yaml` on HA (via Samba `config` share)
3. Run `./deploy-ha-dashboards.sh` to push the YAML file
4. Restart HA: Settings → System → Power → Restart HA
5. Update this index + `CLAUDE.md` file table

---

## Rollback

Revert the YAML file to a previous git commit, then run `./deploy-ha-dashboards.sh`.
